# Installation overview

## What is it?

This section covers installing and upgrading the SQL Agent on Windows machines. Use these pages to plan a new deployment, upgrade an existing agent, run an unattended install, or set up additional agent instances on the same host.

## Pick the right page

| If you are... | Start with |
|---|---|
| Installing the SQL Agent for the first time | [New installation](./new-installation.md) |
| Updating an existing SQL Agent | [Upgrade](./upgrade-installation.md) |
| Deploying the agent through a script or deployment tool | [Silent mode](./silent-mode.md) |
| Running more than one agent on the same machine | [Multiple instances](./multiple-instances.md) |
| Choosing how to obtain the installer | [Installation media](./determine-installation-media.md) |

## In this section

- [Installation media](./determine-installation-media.md) — Available installation media options and how to use each.
- [New installation](./new-installation.md) — Install the SQL Agent for the first time and create its OpCon machine record.
- [Upgrade](./upgrade-installation.md) — Upgrade an existing SQL Agent on a Windows system.
- [Silent mode](./silent-mode.md) — Reference link for silent mode installation.
- [Multiple instances](./multiple-instances.md) — Install additional SQL Agent instances on a single machine.
